# Implementation Plan: SemirPhone Mobile App

**Branch**: `003-semir-phone-app` | **Date**: 2026-04-25 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/003-semir-phone-app/spec.md`

## Summary

SemirPhone is a cross-platform iOS/Android analytics app (Flutter / Dart 3.x) that connects to the existing SemirDashboard backend via a new JWT-authenticated JSON API layer (Sprint 0 backend deliverable). The app is read-only: it displays the same analytics data as the web dashboard — login, home with 8 permission-gated cards, and 7 analytics/chart pages — with pixel-perfect design token parity. Distribution target: Apple App Store + Google Play Store.

## Technical Context

**Language/Version**: Dart 3.x / Flutter 3.x (stable channel)  
**Primary Dependencies**:
- `dio` 5.x — HTTP client with `QueuedInterceptor` for JWT refresh
- `flutter_riverpod` 2.x + `riverpod_generator` — state management
- `flutter_secure_storage` 9.x — Keychain (iOS) / EncryptedSharedPreferences (Android)
- `local_auth` 2.x — Face ID / Touch ID / Android biometric
- `fl_chart` — donut and line charts matching web color palette
- `sentry_flutter` — crash reporting (FR-046, SC-006)
- `golden_toolkit` — visual snapshot generation (FR-032/033)
- `flutter_native_splash` — splash screen
- `intl` — VND number formatting

**Backend**: Django REST Framework + `djangorestframework-simplejwt` added to existing SemirDashboard  
**Storage**: Device secure storage only (no local DB). In-memory page cache via Riverpod.  
**Testing**: `flutter_test` (unit + widget), `integration_test` package (E2E), golden tests (visual)  
**Target Platform**: iOS 14+, Android 8 (API 26)+  
**Performance Goals**:
- Cold start → home ≤ 3s (SC-001)
- Analytics page data ≤ 4s on 10 Mbps network (SC-002)
- Table scroll ≥ 50fps at 200 rows (SC-007)

**Constraints**: Single shared codebase (FR-027), HTTPS only (FR-030), TLS CA pinning (FR-031), tokens never in plaintext (FR-002), no deep links (clarification Q5)  
**Scale/Scope**: 10 screens, 12 API endpoints, 5 permission strings, ≤ 200 rows per table

### Sprint 0 — Backend API (Hard Blocker)

Must complete before mobile Sprint 1. This sprint is owned by the backend developer.

**Implementation:**

1. Add to `SemirDashboard/requirements.txt`: `djangorestframework`, `djangorestframework-simplejwt`, `django-cors-headers`
2. Run `pip install -r requirements.txt`
3. Create `SemirDashboard/App/api/` package: `__init__.py`, `urls.py`, `serializers.py`, `views.py`, `permissions.py`
4. Add `api_urls.py` and wire `path("api/v1/", include("App.api_urls"))` in `SemirDashboard/urls.py`
5. Configure simplejwt in `settings.py`: `ROTATE_REFRESH_TOKENS=True`, `BLACKLIST_AFTER_ROTATION=True`, access lifetime 1h, refresh lifetime 30d
6. Run `python manage.py makemigrations` + `python manage.py migrate` (simplejwt blacklist table)
7. Implement all 12 endpoints per `contracts/api-contract.md` — auth (3) + analytics (5) + charts (3) + shops dropdown (1)
8. Configure `django-cors-headers` in `INSTALLED_APPS` + `MIDDLEWARE`

**Test gate per endpoint (run before marking Sprint 0 done):**

```bash
# For each of the 12 endpoints, the following must pass:

# 1. POST /api/v1/auth/token/ — login
curl -X POST http://localhost:8000/api/v1/auth/token/ \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"<pw>"}' | python -m json.tool
# PASS: response contains access, refresh, permissions[]

# 2. POST /api/v1/auth/token/refresh/ — silent refresh
# PASS: returns new access token; old refresh token is invalidated (check DB blacklist)

# 3. POST /api/v1/auth/logout/ — revoke
# PASS: returns 205; subsequent use of refresh token returns 401

# 4–12. Analytics + chart endpoints
# PASS criteria per endpoint:
# - Returns 200 with correct JSON shape matching contracts/api-contract.md
# - Returns 401 without auth header
# - Returns 403 with valid token that lacks the required permission
# - Returns 400 for invalid date params (date_from > date_to)
# - VND values are pre-formatted strings (e.g. "1,234,567" not 1234567)
# - Percentage values include 2 decimal places (e.g. "45.98" not "46")
```

**Regression check:** Run `python manage.py test tests -v 2` in SemirDashboard — all existing web tests must still pass (DRF layer must not break existing HTML views).

**Exit Criteria — Sprint 0 is DONE when:**
- [ ] All 12 endpoints return correct JSON shapes when called with valid credentials
- [ ] All 12 endpoints return 401 without auth, 403 with wrong permission
- [ ] Token rotation verified: refresh token is blacklisted after use
- [ ] All pre-existing SemirDashboard tests pass (no regression)
- [ ] `python manage.py runserver` starts without error with new packages
- [ ] API contract document (`contracts/api-contract.md`) updated with any deviations found during implementation

## Key Design Decisions

### JWT Refresh Interceptor (Critical)

The `auth_interceptor.dart` implements:
```
QueuedInterceptor.onError:
  if (error.response?.statusCode == 401 && !_isRefreshing):
    _isRefreshing = true
    try:
      newTokens = await authService.refresh()
      retry all queued requests with new token
    catch:
      route to login ("session expired")
    finally:
      _isRefreshing = false
  else if (error.response?.statusCode == 401 && _isRefreshing):
    queue this request's retry behind the in-flight refresh
```

This is the concurrent 401 edge case from the spec. Testing: set access token lifetime to 5s in debug, fire 3 parallel requests after expiry, verify only 1 refresh call hits the backend.

### Design Token System

`app_colors.dart` defines:
```dart
const Color primary = Color(0xFF0D6EFD);           // #0d6efd
const Color allTimeCardBg = Color(0x14FF8C00);      // rgba(orange, 0.08)
const Color periodCardBg = Color(0x170D6EFD);       // rgba(primary, 0.09)
const Color textDark = Color(0xFF212529);           // var(--text)
const Color textMuted = Color(0xFF6C757D);
const Color navBg = Color(0xFF1A1D23);             // var(--nav-bg) for dark tabs
```

These mirror the web CSS tokens exactly. All widgets use these constants — no hardcoded hex anywhere except `app_colors.dart`.

### Sticky-Column Table

`data_table_widget.dart` uses a `Row` of two `SingleChildScrollView`:
- Left column: fixed-width, vertical scroll only (the label column — sticky)
- Right columns: synchronized horizontal + vertical scroll (data columns)

Both scroll controllers are linked vertically. This achieves sticky first column on any Flutter version without a third-party package.
